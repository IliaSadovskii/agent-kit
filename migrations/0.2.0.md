# Migrating to 0.2.0 — from a hand-copied kit to the installer

Only relevant for a project that got the kit by copying files before this repository existed.
A fresh `install.sh install` needs none of this.

## What changes

The kit is now installed and updated by `install.sh`, which tracks every kit-owned file in
`.agent-kit/kit.lock`. Without that lock the installer cannot tell an untouched file from one you
edited, so the first run treats *every* differing file as a conflict — safe, but noisy.

## Steps

1. Commit or stash your work; the run should be reviewable as a diff.
2. Install over the existing copy:

   ```bash
   bash install.sh install --from /path/to/agent-kit
   ```

3. Expect conflicts for any file you had customized, each with a `<file>.kit-new` copy beside it.
   For each one, decide:
   - the release version is fine → replace your file with the `.kit-new` copy;
   - your version is project-specific → move the rule into `.agent-kit/project/instructions.md` and
     take the release version;
   - the change is generally useful → send it to the kit repository, then take the release version.
4. Delete the leftover `.kit-new` files and run the update once more so the lock records the
   resolved state.
5. Check `.agent-kit/project/manifest.yml` still holds your real values — the installer only writes
   `kit_version` there, and never overwrites the file.
6. Run `.agent-kit/scripts/validate.sh`, then commit.

## Files that moved

Nothing moved inside a project. What changed is where the kit is authored: `.claude/settings.json`
and `.codex/hooks.json` are now shared project files that the installer only adds its SessionStart
hook to, rather than kit-owned files it replaces. If you had customized either, your version stays.
